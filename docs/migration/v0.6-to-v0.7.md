# Migrating from v0.6 to v0.7

## sync.batch_size → sync.batch_config.size

The `batch_size` key has been moved under `batch_config` to support additional batching options (e.g., `batch_config.timeout`).

### Old config

```yaml
sync:
  batch_size: 500
```

### New config

```yaml
sync:
  batch_config:
    size: 500
```

### Why

`batch_size` was moved under `batch_config` to provide a unified namespace for all batching-related options. This allows future extensions like `batch_config.timeout` without requiring separate top-level keys.

### Migration steps

1. Locate all `sync:` sections in your YAML files that have `batch_size` at the top level
2. Create a `batch_config:` section if it doesn't exist
3. Move the `batch_size` value to `batch_config.size`
4. Test with `drt validate` to ensure the new config is recognized
5. Run `drt run --dry-run` to verify behavior is unchanged
